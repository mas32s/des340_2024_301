In our first exercise, we will design a so called one pager site. One pagers are websites from a single page.
Most of them has a single purpose: to make people to do something. It is Call to action.

How to do that?

Select a project from kickstarter.com. The purpose of the website is to make people to back the project.

Here are some guidelines:
  1. Select a real project, not a dream. This is a dream: https://www.kickstarter.com/projects/katescreations/freeze-dried-sweet-treats?ref=discovery_category This is a project: https://www.kickstarter.com/projects/klos/carbon-fiber-concert-ukuleles What is the difference? The ukulele is ready, there is a target audience and a price for the product.
  2. Select a project with considerable content. Example: https://www.kickstarter.com/projects/clickpackpro/snappack-travel-commute-and-anti-theft There is a lot of explanation about the project and high resolution images from the product without any graphics and text on them. Tip: check the official website of the product for high resolution images
  3. Select a right kind of project There are two kind of products, like iPhone (https://www.apple.com/iphone/) and exquisite tea (https://exquisiteleaves.com/). The iPhone is designed to sell itself, it look luxury and elegant and expensive. The tea itself looks like some suspicious weed. So, if you select the first kind, the images of the product maybe enough to make the webdesign. If you want to sell tea, the images of the tea don't work. Yo have to get images of a freshly made tea and happy people consuming it. Or make drawings, whatever. But you have to make some content.
  4. Save the content of the selected project Once you selected your product, save all the text and images from the browser (there is a save webpage command in every browser). Remember: The project on the Kickstarter can be cancelled any time, so be sure, you have the content.
Wireframing Wireframes are blueprints for websites. Websites are too complex to design them at once, like a print. Wireframes are drafts, what show the structure and the operation of thw website but not the design. They are easy to create and change, so ideal way to make your ideas clear. Today we show thwm to clients. That's why we don't use any real design elements (colors, images, fonts and typography). We have to make clear, that they are blueprints, not graphics designs. Steps Inspire yourself! Search onepagers what solve the same or similar problems than yours. (onepagerlove.com is a good start) Prepare your wireframe in Figma. You will need some real screen size setup (a desktop computer and a moblile) and some components. Good idea to make components for image and video placeholders, for title, subtitle and bread text. You will need a button, and a sticky note is also useful. (My example wireframe's link is below) Decide your basic margins. You will need two sets: one for keeping the content in center on big desktop screens and an other what serves as margin for text on mobile. Visual Design

Visual design has three fundamental elements:

  1. Logo
  2. Colors
  3. Font
Logo is usually given. If not, yo should create one. Or at least use the project name with some considerable typography.
